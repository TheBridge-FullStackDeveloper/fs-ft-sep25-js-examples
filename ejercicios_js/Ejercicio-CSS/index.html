POSITION

JavaScript nació en 1995 en Netscape, cuando Brendan Eich creó en apenas diez días un lenguaje que originalmente se llamó Mocha. Más tarde pasó a llamarse LiveScript, porque la idea era que fuese un lenguaje ligero para interactuar con la web dentro del navegador Netscape Navigator. Sin embargo, justo en ese momento, el lenguaje Java de Sun Microsystems estaba en pleno auge mediático. Para aprovechar esa popularidad y hacer que la gente lo relacionara rápidamente con algo potente y moderno, Netscape y Sun decidieron rebautizar LiveScript como JavaScript.

El nombre generó mucha confusión porque Java y JavaScript en realidad son lenguajes totalmente diferentes: Java es un lenguaje compilado, fuertemente tipado y orientado a objetos, mientras que JavaScript es interpretado, dinámico y más flexible. Pero la jugada de marketing funcionó: al llevar “Java” en el nombre, JavaScript fue adoptado rápidamente por los desarrolladores web, y poco a poco se convirtió en el estándar para la interactividad en el navegador.

A finales de los años noventa, cada navegador tenía su propia versión de JavaScript, lo que creaba incompatibilidades enormes. Para resolverlo, se estandarizó bajo el nombre de ECMAScript, definido por la organización ECMA International. Desde entonces, el lenguaje ha evolucionado constantemente, incorporando nuevas funcionalidades como las arrow functions, las promesas, el async/await o los módulos modernos, pero siempre conservando el nombre JavaScript en la práctica cotidiana.

Así que, aunque hoy en día el nombre JavaScript pueda parecer fruto de una confusión, en realidad fue un movimiento estratégico: gracias a esa decisión de marketing, el lenguaje ganó tracción, se convirtió en el motor de la web moderna y sigue siendo uno de los lenguajes más usados del mundo. En resumen, JavaScript no se llama así por ser una versión de Java, sino porque en su origen necesitaba subirse a la ola de popularidad de Java y conseguir atención en un ecosistema que estaba naciendo.

Uno de los relatos más repetidos sobre JavaScript es que fue creado en apenas diez días. En 1995, Brendan Eich trabajaba en Netscape y le pidieron que diseñara un lenguaje de scripting que pudiera ejecutarse dentro del navegador. La urgencia era enorme: necesitaban un lenguaje ligero que complementara a Java y que permitiera dar algo de dinamismo a las páginas web, que hasta entonces eran completamente estáticas. Eich lo hizo en cuestión de días, y esa rapidez marcó tanto la identidad del lenguaje como la percepción que tuvieron muchos desarrolladores durante años.

Durante mucho tiempo, JavaScript fue visto como un “juguete”. Se usaba para cosas pequeñas como abrir alertas, cambiar colores al pasar el ratón o validar un formulario antes de enviarlo. La comunidad de desarrolladores “serios” lo despreciaba, porque lo veían limitado, mal diseñado y lleno de rarezas. De hecho, existía un chiste recurrente: “Java es a JavaScript lo que una cafetera es a un café instantáneo”. Se le consideraba el patito feo de la programación.

Sin embargo, poco a poco el lenguaje empezó a ganar terreno. Con la llegada de AJAX a principios de los 2000, JavaScript pasó de ser un juguete a una herramienta clave para crear aplicaciones web más interactivas, como Gmail o Google Maps, que mostraron al mundo que las páginas podían comportarse casi como aplicaciones de escritorio. Después llegó la explosión de librerías y frameworks como jQuery, Angular, React o Vue, que terminaron de consolidar a JavaScript como el corazón de la web moderna.

Hoy, aquellas críticas parecen casi irónicas: el lenguaje que nació como un experimento apresurado y que fue odiado durante años es ahora uno de los pilares del desarrollo tecnológico. Gracias a Node.js, incluso se usa en el servidor, rompiendo la barrera de “solo navegador” y permitiendo a millones de programadores trabajar con un mismo lenguaje en front y back. Además, con la llegada de TypeScript, ES6 y todas las mejoras modernas, JavaScript se ha convertido en un ecosistema robusto y maduro.

Lo curioso es que, a pesar de sus defectos de origen, de los chistes y de los detractores, JavaScript se mantiene como el lenguaje más popular del mundo según numerosas encuestas y rankings. Lo que empezó en diez días como un parche para un navegador terminó siendo la columna vertebral de Internet. Esa es, quizá, la mayor leyenda de todas: un lenguaje que nadie respetaba se volvió imprescindible.
